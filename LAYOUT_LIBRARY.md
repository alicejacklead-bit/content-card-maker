# Layout Library

## Purpose

This library defines reusable content-card layouts for `content-card-maker`.

It does not choose a layout.

Layout selection is owned by `VISUAL_ENGINE.md`.

This file describes how each layout works so the selected structure can be
applied consistently and expanded later.

---

## Usage Rule

When `VISUAL_ENGINE.md` selects a layout:

1. Use the layout whose information flow best matches the source content.
2. Keep one dominant reading pattern per card or card sequence.
3. Prefer mobile-first scanning over desktop-style complexity.

---

## Layout Format

Each layout uses the same fields:

- Applicable Scenarios
- Page Structure
- Information Density
- Reading Order
- Best Platforms

---

## Hero

**Applicable Scenarios**  
Cover pages, product openers, single big idea, headline-led announcements.

**Page Structure**  
Large headline, optional subheadline, one dominant visual anchor, small supporting cue.

**Information Density**  
Low.

**Reading Order**  
Headline first, visual second, supporting text last.

**Best Platforms**  
Xiaohongshu, Douyin, WeChat Feed.

---

## Checklist

**Applicable Scenarios**  
Tips, habits, reminders, do and do not lists, quick practical content.

**Page Structure**  
Title area, vertical checklist items, optional summary or tag block.

**Information Density**  
Medium.

**Reading Order**  
Top-down scan, item by item.

**Best Platforms**  
Xiaohongshu, WeChat Feed.

---

## Numbered List

**Applicable Scenarios**  
Lists, ranked points, key takeaways, step collections, core principles.

**Page Structure**  
Title, numbered modules, short explanation under each number.

**Information Density**  
Medium.

**Reading Order**  
Sequential from 1 to n.

**Best Platforms**  
Xiaohongshu, Douyin, WeChat Feed.

---

## Comparison

**Applicable Scenarios**  
Before versus after, option A versus B, myth versus fact, old way versus new way.

**Page Structure**  
Two-column or split-card comparison with shared comparison criteria.

**Information Density**  
Medium.

**Reading Order**  
Left-right or top-bottom pair comparison, then conclusion.

**Best Platforms**  
Xiaohongshu, WeChat Feed, Douyin.

---

Linear time markers with step summaries and optional milestone highlight.

**Information Density**  
Medium.

**Reading Order**  
Chronological path.

Xiaohongshu, WeChat Feed.

## Step-by-Step

**Applicable Scenarios**  
How-to guides, workflows, procedural tutorials, onboarding instructions.

**Page Structure**  
Step labels, action description, optional tips or warnings for each step.

**Information Density**  
Medium to high.

**Reading Order**  
Strictly sequential.

**Best Platforms**  
Xiaohongshu, WeChat Feed, Douyin.

---

## Flowchart

**Applicable Scenarios**  
Decision paths, workflows, systems, process logic, tool pipelines.

**Page Structure**  
Nodes, connectors, optional branch labels, clear start and end points.

**Information Density**  
Medium to high.

Follow connectors from entry node to outcome.

**Best Platforms**  
WeChat Feed, Xiaohongshu, Douyin when simplified.

**Information Density**  
Medium.

**Reading Order**  
Linear stage progression.

**Best Platforms**  
WeChat Feed, Xiaohongshu.

**Applicable Scenarios**  
Feature lists, grouped tips, category clusters, short repeated points.

**Page Structure**  
Uniform card grid with title, icon, and short body in each cell.

**Information Density**  
Medium.
